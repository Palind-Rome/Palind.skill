# Design Notes

这个 Skill 把 Palind 的三篇文章拆成两层：

- `work.md`: 可执行写作方法，包括文章结构、节奏、素材域、生成流程和自检规则。
- `persona.md`: 表达 DNA、心智模型、决策启发、反模式、诚实边界和 agentic protocol。

## 核心模型

### 物件回声

文章从一个小物件或短句进入，并在结尾回收它。物件不是装饰，而是组织记忆和情绪的锚点。

### 小系统里的面子和秩序

学校、课堂、食堂、考试、前台、互联网和 LLM agent 都被写成小系统。个人的尴尬经常表现为系统运行中的小故障。

### 幼稚误解比真相更能解释童年

童年的误读、谣言和错觉被保留，而不是立即纠正。成年知识后来进入文本，形成错位和回声。

### 技术回头照亮过去

LLM、agent、编程语言和终端不是单独的科技话题，而是回看机房、Pascal、作业、课堂和考试的新语言。

## 生成原则

- 先写具体场景，再写观点。
- 先让读者笑，再让情绪落地。
- 技术名词必须服务于人的处境。
- 结尾回到开头，但不能只是重复开头。
- 不复述源文情节，不冒充作者本人。
